# Example: Decision Brief for Internal Reference Gathering

## User request

我要为一篇非商业草稿收集视觉参考图和反应 GIF，只用于内部选题和风格讨论。不要只给我官方图库，也不要按商业发布标准卡住。

## Expected PavedPath behavior

Use Decision Brief. The user's current goal is internal research and inspiration, not public release or commercial delivery. Do not treat publication-grade licensing as the default gate. The useful path is usually a fast, traceable reference workflow built from public examples, forum/social posts, screenshots, existing memes/reaction GIFs, and source URLs, with a later clearance/replacement step only if the material moves into public or commercial output.

Use Agent Reach first when available because the answer depends on current public examples and community practice.

Expected evidence surfaces:

```text
Hard boundaries:
- Do not collect private, leaked, paywalled, credential-protected, or clearly unlawful material.
- If the output will be published or sold later, rights/permission checks become a separate release step.

Real-world execution evidence:
- creator workflows for moodboards, swipe files, reaction GIF banks, and visual reference packs
- forum/social/video examples showing how people collect, label, and reuse references internally
- failure retrospectives where teams lost track of sources or mixed internal references into publishable assets
- official stock/library sources only when the user needs release-ready assets now
```

Expected answer shape:

```text
Recommendation: Build an internal reference pack first, not a publication-ready asset library.
Why this path wins: faster exploration, wider coverage of real styles, lower setup cost, and source traceability without premature licensing work.
Options:
- Internal reference pack from public examples: best for ideation; save image/GIF, source URL, platform, observed use, and notes; mark internal-only.
- Official stock/library-only route: use when the material will ship publicly with minimal redesign; slower and narrower for early ideation.
- Generate-from-scratch route: useful after the direction is chosen, not as the first discovery step if the goal is to understand existing visual language.
Avoid / delay: Do not spend the first pass clearing every asset for commercial use when the current artifact is an internal draft. Do not publish collected references directly.
Next action: Collect 30-50 references across 3-5 platforms, dedupe them, tag by use case, and keep a release column: internal-only / replace before publish / cleared for publish.
```
