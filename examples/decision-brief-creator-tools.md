# Example: Decision Brief

## User request

I want to build a lightweight content idea database. Should I use Notion, Airtable, or a spreadsheet?

## Expected PavedPath behavior

Use Decision Brief. Compare paths by fit and maintenance burden instead of writing a full research report.

```text
Recommendation: Start with a spreadsheet if the workflow is solo and mostly text/list tracking; move to Notion when you need richer briefs and references; use Airtable only when structured fields, views, and collaboration become important.

Options:
- Spreadsheet: fastest, lowest maintenance, best for validating the habit.
- Notion: better for idea notes, scripts, references, and lightweight publishing workflows; can become messy if databases are overdesigned.
- Airtable: strongest for structured pipelines and multi-view production tracking; higher setup cost.

Avoid / delay: Do not start with a complex database schema before you know which fields you actually reuse.
Next action: Track 20 ideas in the simplest tool for one week, then promote only the fields you repeatedly use.
```
